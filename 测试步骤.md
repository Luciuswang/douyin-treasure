## 🧪 测试步骤

### 步骤1：测试健康检查端点

**方法A：使用隐私模式（推荐）**
1. 按 `Ctrl + Shift + N` 打开隐私模式
2. 访问：`https://totofun-server007.vercel.app/health`
3. 应该返回：
   ```json
   {
     "status": "ok",
     "timestamp": "...",
     "uptime": 0.xxx,
     "environment": "production"
   }
   ```

**方法B：清除浏览器缓存**
1. 按 `Ctrl + Shift + Delete`
2. 清除"缓存的图片和文件"
3. 重新访问：`https://totofun-server007.vercel.app/health`

### 步骤2：测试注册功能

1. **访问前端页面**
   - 打开：`https://luciuswang.github.io/douyin-treasure/`
   - 使用隐私模式或清除缓存后访问

2. **尝试注册**
   - 点击"登录/注册"按钮
   - 切换到注册表单
   - 填写信息：
     - 用户名：`testuser123`
     - 邮箱：`test@example.com`
     - 密码：`test123456`
   - 点击"注册"按钮

3. **检查结果**
   - 打开浏览器开发者工具（F12）
   - 查看 Console 标签：
     - ✅ 应该看到 API 配置信息
     - ✅ 应该看到请求发送到 `https://totofun-server007.vercel.app/api/auth/register`
   - 查看 Network 标签：
     - ✅ 请求状态应该是 200 或 201（成功）
     - ✅ 响应应该包含用户信息和 token

### 步骤3：如果SSL错误仍然存在

**临时解决方案**：
1. 点击浏览器地址栏的"高级"或"Advanced"
2. 点击"继续访问"或"Proceed to site"
3. 这会让浏览器忽略SSL证书错误（仅用于测试）

**注意**：这只是临时方案，用于测试。如果注册功能能正常工作，说明服务器没问题，只是SSL证书显示问题。
